Features of C# 7.0

Definition/Concept

C# 7.0 introduced language enhancements for data consumption, code simplification, and performance: tuples, pattern matching, out variables, literal/operators improvements, and more.

Detailed Explanation

  • Out Variables: Declare out args inside method calls for cleaner code.
  • Tuples: Easily return multiple values.
  • Pattern Matching: Flexible condition checks and assignments.
  • Digit separators & Binary literals: Improved number readability.
Analogy: Upgrade your programming "toolbox" for easier, more efficient code.

Code Example

// Out Variables
if (int.TryParse("123", out int result)) {
    Console.WriteLine(result); // result is in-scope
}

// Tuple
public (string, int) GetPerson() => ("John", 30);

// Pattern matching
if (obj is string s) Console.WriteLine(s.Length);
Exam Notes:
Major features: Out variables, tuples, patterns. Improves readability and performance.

  • What is the benefit of out variables?

C# Compilation and Execution

Definition/Concept

C# code is compiled to intermediate language (MSIL), then executed by the Common Language Runtime (CLR) using Just-In-Time (JIT) compilation.

Detailed Explanation

C# Source (.cs)
    ↓
csc.exe -> MSIL (.exe/.dll)
    ↓
CLR + JIT -> Native machine code
Analogy: Like translating a book—first to a universal language, then to local.

Code Example

using System;
class Program {
    static void Main() {
        Console.WriteLine("Hello World!");
    }
}
Exam Notes:
Process: Src → MSIL → Native. Key: csc, CLR, JIT. Provides platform independence.

General Structure of a C# Program

Definition/Concept

Organized as: using directives, namespace, class, and Main entry method.

Detailed Explanation

  • Using statements - import namespaces
  • Namespace - code organization
  • Class - object template
  • Main - entry point
Analogy: Like a blueprint: tools list, address, room, entrance.

Code Example

using System;
namespace App {
    class Program {

    }
}
Exam Notes:
Main method is entry. Common signatures: static void/int Main(...). Structure: using → namespace → class → Main.

Data Types in C#

Definition/Concept

Defines the type of value stored: Value, Reference, (Pointer - unsafe code).

Detailed Explanation

  • Value types: int, float, bool, char; stored on stack.
  • Reference types: string, object, class, array; stored on heap.
Analogy: Different containers in a warehouse, some hold actual goods, some hold address tags.

Code Example

int age = 25;
string name = "John";
int[] numbers = { 1, 2, 3 };
int a = 10; int b = a; b = 20; // Value types – separate copies
int[] arr1 = {1,2,3}; int[] arr2 = arr1; arr2 = 100; // Reference types - shared data
Exam Notes:
Value types on stack, reference types on heap. Use GetType(), sizeof() for info.

Boxing and UnBoxing

Definition/Concept

Boxing converts value type to reference (object); unboxing extracts value type from object.

Detailed Explanation

  • Boxing: Implicit conversion, stores value on heap.
  • Unboxing: Explicit conversion, retrieves value from object.
Analogy: Wrapping/unwrapping a gift box.

Code Example

int n = 123;
object boxed = n;        // boxing
int unboxed = (int)boxed;// unboxing
Exam Notes:
Boxing creates overhead, used in collections expecting objects. Unboxing needs type check.

String vs StringBuilder

Definition/Concept

String is immutable; StringBuilder is mutable, more efficient for repeated changes.

Detailed Explanation

  • String: Creates new object each modification.
  • StringBuilder: Single object, changes content in place.
Analogy: Pen on paper (String) vs Whiteboard (StringBuilder).

Code Example

string s = "";
for (int i=0; i<5; i++) s += i; // many objects
StringBuilder sb = new StringBuilder();
for (int i=0; i<5; i++) sb.Append(i); // single object
Exam Notes:
Use StringBuilder for loops/modifications; String for simple assignments.

Default Parameters, Named Parameters

Definition/Concept

Default values for method parameters; named arguments for flexible calling.

Detailed Explanation

  • Default parameters: placed at end; must be compile-time constants.
  • Named parameters: specify arguments by name, in any order.
Analogy: Forms with preset options (defaults), fill fields in any order (named).

Code Example

void Print(string name, int age=18, string city="Unknown") {
    Console.WriteLine(name);
}

Print("John", age:22); // Uses default city
Print(name:"Jane", city:"Paris"); // Uses default age
Exam Notes:

Parse() vs TryParse() vs Convert Class Methods

Definition/Concept

Parse(): direct conversion, throws exception. TryParse(): safe conversion, returns bool. Convert: versatile and handles null.

Detailed Explanation

  • Parse(): for trusted input, throws error if fails.
  • TryParse(): for user/unknown input, safe, no exceptions.
  • Convert: works with mixed types, handles null, slightly slower.
Analogy: Parse: "Must succeed or error", TryParse: "Let me know if you can't", Convert: "Flexible translator."

Code Example

// Parse
int n = int.Parse("123"); // Throws if invalid

// TryParse
bool ok = int.TryParse("abc", out int value); // ok=false, no exception

// Convert
int val = Convert.ToInt32(null); // returns 0, no exception
Exam Notes:
Use TryParse with user input, Convert for flexible handling, Parse for guaranteed valid input.
